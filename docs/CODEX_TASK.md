# Codex 开发任务：CRSwNP 术后智能风险分层与随访平台

## 目标
从零开发一个可本地运行的医学科研概念验证平台，用于展示 CRSwNP 术后科研病例管理、模拟风险分层、随访管理和科研统计。

## 技术栈
- 前端：React + TypeScript + Vite + Ant Design + ECharts
- 后端：FastAPI + SQLAlchemy + Pydantic
- 数据库：SQLite
- 启动：Docker Compose，同时提供普通本地启动方式
- 测试：基础 API 测试、前端构建测试

## 约束
- 仅使用模拟患者、模拟参数和模拟结果。
- 禁止录入真实姓名、身份证号、手机号、住院号、门诊号等可识别信息。
- 禁止接入 HIS、EMR、PACS、LIS 或其他真实医疗系统。
- 所有风险结果必须标注：仅用于科研概念验证，不可直接用于临床诊疗决策。

## 页面
1. 登录页
2. 首页仪表盘
3. 患者列表页
4. 新增与编辑患者页
5. 患者详情与风险评估页
6. 随访管理页
7. 科研统计页
8. 数据字典页

## 数据模型
至少包含：
- users
- patients
- risk_assessments
- followup_plans
- followup_records

## 模拟风险模型
建立独立配置文件，例如：
`backend/app/config/risk_model.json`

模型版本：
`CRSwNP-DEMO-v0.1`

使用模拟 Logistic 回归系数计算风险概率，阈值可配置：
- 低风险：<20%
- 中风险：20%–49.9%
- 高风险：≥50%

## 演示数据
自动生成不少于 60 例模拟患者，覆盖：
- 低风险、中风险、高风险
- 已完成随访、待随访、已逾期
- 鼻喷雾、大容量鼻腔冲洗
- 不同症状严重程度与随访结局
